Git Set Up

Name und E-Mail-Adresse festlegen


$ git config --global user.name "Name"
$ git config --global user.email "E-Mail Adresse"

E-Mail gleich der E-Mail Adresse bei GitHub!

Grundsätzliche Begriffe / Befehle

  • init → Eigenes unabhängiges lokales Repository erstellen
  • clone → Verknüfte kopie eines remote Repositries erstellen
  • status → Übersicht lokaler Änderungen anzeigen
  • add → Änderungen zum Übernehmen markieren
  • commit → Änderungen dauerhaft mit Log-Eintrag übernehmen
  • pull → Änderungen an remote Repository senden
  • push → Lokales Repository aktuallisieren

Demo (Clonen und Commiten)


$ git clone git@github.com:fh-koeln/wba1-2012.git
$ cd wba1-2012
$ ...
$ git add .
$ git commit -m "Neuer Branch für Git basics"
$ git push

Branchen

  • Problem: Master ist keine Spielwiese
  • Lösung: Eigener, temporärer „Entwicklungszweig“
  • → Lokale und unabhängig
  • → Pushen optional
  • → WBA1 Namenskonvention beachten

Demo (branchen)


# Branch erstellen:
$ git branch git-basics
$ git checkout git-basics
$ ...

# Branch verwenden:
$ git checkout master
$ git checkout git-basics
$ ...
					

$ cp -r yyyy-mm-dd-lehreinheit/ 2012-12-06-git-basics/
$ git add .
$ git commit -m "Neuer Branch für Git basics"
$ git push origin git-basics
Branches auf GitHub: master git-basics

Mergen

Ergebnisse zusammenführen:

  • Beim parallelem Arbeiten (gleicher Branch)
  • Ergebnisse veröffentlichen (eigener Branch → master)
  • Branch aktualisieren (master → eigenen Branch)
  • Pull requests

Demo (mergen)


# Automatisch beim parallelem Arbeiten

# Branches hin- und her mergen, hier git-basics -> master
$ git branch
* master
  git-basics

# Implizierter Commit!
$ git merge git-basics
$ git push

Konflikte

  • Wann passieren sie?
  • → bei git pull
  • → bei git merge
  • Wenn Änderungen die nicht automatisch mergebar sind.

Demo (Konflikte)


alice $ vim demo.txt
alice $ git commit -am "Alice commit"
alice $ git push

bob $ vim demo.txt
bob $ git commit -am "Bobs commit"
bob $ git pull                  # KONFLIKT
bob $ vim demo.txt              # KONFLIKT manuell lösen
bob $ git commit -am "Solved!"
bob $ git push
