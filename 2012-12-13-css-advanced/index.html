1. Positionierung

  • Art der Positionierung
  • Positionierung anhand von Abständen
  • Überlagerung von Elementen

1.1 Arten der Positionierung

Insgesamt 4 Arten der Positionierung

  • position: static;
  • position: relative;
  • position: absolute;
  • position: fixed;

1.1.1 Statische Positionierung

  • Standard, falls nicht anders angegeben
  • Angezeige wie sie im HTML vorkommen

1.1.2 Relative Positionierung

  • können verschoben werden
  • nachfolgende Elemente verhalten sich als wäre das Element noch an der ursprünglichen Stelle
  • können andere Elemente überlagern

1.1.3 Absolute Positionierung

  • werden aus der normalen Reihenfolge herausgelöst
    • Position wird nicht durch andere Elemente beeinflusst
    • Position beeinflusst andere Elemente nicht
  • Position wird aus der linken, oberen Ecke des Eltern-Elements gemessen
  • können andere Elemente überlagern

1.1.4 Fixe Positionierung

  • wird im Bezug auf das Browserfenster positioniert
  • bewegt sich nicht mit, wenn gescrollt wird

1.2 Ausrichten mit Abständen

nur mit relativer, absoluter, fixer Position

  • horizontaler Abstand
    • left: value;

      • oder

    • right: value;
  • vertikaler Abstand
    • top: value;

      • oder

    • bottom: value;

1.2.1 Beispiel

.element {
	position: relative;
	top: 100px;
	left: 100px;
}
.element2 {
	position: absolute;
	bottom: 0px;
	right: 0px;
}

1.3 Überlagerung

Was wenn Elemente übereinander liegen?

  • Zuletz genanntes Element im HTML Code liegt vorne
  • Steuerbar über 
    z-index: value;
  • Element mit höherem z-index liegt vorne

2. Dokumentenfluss

  • "fließende" Layouts
  • flexible Seiten, die sich anpassen
  • Spalten Layouts

2.1 float

    float: left; oder float: right;
  • lassen Elemente nach links oder rechts "fließen"
  • Element und folgende werden soweit wie möglich nach links bzw. rechts gerückt
  • ein gefloatetes Element verliert typische Block Eigenschaften

2.2 clear

    clear: left; oder clear: right; oder clear: both;
  • beendet den float Zustand

3. Pseudoklassen und Pseudoelemente

  • Pseudoklassen für Verweise und andere Elemente
  • Pseudoklassen für Absätze
  • Pseudoelemente für automatisch generierten Inhalt
  • Weitere Pseudoklassen

Pseudoklassen für Verweise und andere Elemente

  • :link
  • :visited
  • :focus
  • :hover
  • :active

Pseudoklassen für Absätze

  • :first-child
  • :first-line
  • :first-letter
Seit CSS3 auch noch:
  • :last-child
  • :nth-child

:first-child

Angenommen die Listen-Elemente einer Auflistung der neusten Blog-Artikel haben alle folgende border-top CSS-Eigenschaft:

ul#recent-posts li {
border-top: 1px solid #333;
}

Nun wollen wir die "top-border" wegbekommen beim obersten Element:

ul#recent-posts li:first-child { border-top: none;}

:last-child

Nehmen wir nun die "bottom-border" vom letzten Element weg:

ul#recent-posts li:last-child {
border-bottom: none;
}

:nth-child

Für das "nth-child" nehmen wir ein Argument hinzu, wie z.B. (odd) oder (even).

tr:nth-child(odd) td {
background: #C1B49A;
}

Man kann aber auch jedes 3te, 4te...n'te Element anstprechen

 *:nth-child (3n+1){...}

Pseudoelemente für automatisch generierten Inhalt

  • :before
  • :after
  • fügt vor oder nach einem Element automatisch Inhalt ein

4. Kombinatoren

  • Nachfahren-Kombinator (Abhängiger Selektor)
  • Kind-Kombinator (Kindselektor)
  • Benachbarte-Geschwister-Kombinator (Nachbarselektor)
  • Allgemeiner-Geschwister-Kombinator (Geschwisterselektor)

Nachfahren-Kombinator
(Abhängiger Selektor)

  • wählt alle Kinder und Kindeskinder (Nachfahren) eines Elements aus

Beispiel zu Nachfahren-Kombinator

#content a {
	text-decoration: underline;
}

<body>
   <div id="header">
      Hier ist der Header!!! <a href="www.linkzumziel.de">Link im Header</a>
   </div>
   <div id="content">   
      <a href="www.linkzumziel.de">Hier der Linktext</a> 
      <p>
         <a href="www.linkzumziel.de">Hier der Linktext</a> 
      </p>
   </div>
</body>

Kind-Kombinator (Kindselektor)

  • direkte Eltern-Kind-Beziehung

Beispiel zu Kind-Kombinator

div > p {
	color: white;
	background: red;
}

<body>
   <p> ... </p>
   <div id="irgendwas">
      <p> ... </p>
   </div>
   <p> ... </p>
</body>

Benachbarte-Geschwister-Kombinator

  • erfasst Elemente, die unmittelbar aufeinander folgen

Beispiel zu Benachbarte-Geschwister-Kombinator


h1 + p {
	font-weight: bold;
}


<body>
   <h1>Die H1 Überschrieft!!!</h1>
   <p>Hier kommt ein Text. Vermutlich mal wieder Lorem Ipsum</p> 
   <p> ... </p>
   <h2>Die H2 Überschrift!!!</h2>
   <p> ... </p>
   <p> ... </p>
</body>

Allgemeiner-Geschwister-Kombinator

  • es werden alle Elemente angesprochen, die im Elementbaum in derselben Ebene auf ein Element folgen
  • unabhängig davon, ob sich zwischen den Elementen weitere Elemente befinden
  • mit CSS3 neu eingeführt

Beispiel zu Allgemeiner-Geschwister-Kombinator


h1 ~ p {
	font-weight: bold;
}


<body>
   <p>Erster Absatz.</p>
   <h1>Der Geschwisterselektor</h1>
   <p>Zweiter Absatz.</p>
   <hr>
   <p>Dritter Absatz.</p>
</body>
