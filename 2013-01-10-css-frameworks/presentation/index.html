Was ist SASS bzw. SCSS

SASS - Syntactically Awesome Stylesheet

SCSS - Sassy Cascading Stylesheet

 

  • SASS ist ein auf Ruby basierendes Framework zur Vereinfachung der Entwicklung von CSS-Layouts und wurde bereits 2007 von Hampton Catlin, Nathan Weizenbaum und Chris Eppstein eingeführt
  • SASS ist eine Metasprache zu CSS
    • d.h. SASS-Dateien werden in standard CSS-Dateien übersetzt
  • Erweitert CSS3 mit "programmiertechnischen" Funktionen wie:
    • Variablen
    • Nesting (Verschachtelung)
    • Mixins
    • Vererbung
  • SASS fungiert Serverseitig bzw. per Tools auf dem Desktoprechner

Warum SASS, SCSS & Co benutzen

"Das Erstellen und Warten umfangreicher Stylesheets ist aufwändig und zeitraubend"
Mathias Schäfer - Webkrauts.de

  • Angaben können zentral definiert werden.
    (z.B. Farben, Abstände etc.)
  • SASS kann rechnen
    (z.B. Farben und Abstände relativ zu einem Wert berechnen)
  • Übersichtlicher und strukturierter
    • Selektoren können andere Eigenschaften erben
    • Selektoren können geschachtelt werden
  • Folglich: erhöhte Wartbarkeit

Wo und wie wird SASS/SCSS eingesetzt?

 

  • Im privaten Bereich findet SASS immer mehr und mehr Anwender
  • Auf Webseiten von Unternehmen mit tausende von CSS-Codezeilen
  • Mit diversen Tools die auf einen Ordner "horchen"

Unterschiede zwischen
SASS und SCSS

SASS gibts es in zwei Dialekten: SASS und SCSS

  • SASS - Syntactically Awesome Stylesheet
    • Ältere Variante
    • Verzichtet auf Klammern und Semikolons
    • Achtet stattdessen auf die Einrückung des Codes
    • Dateiendung: .sass
  • SCSS - Sassy Cascading Stylesheet
    • Neuere und zumeist genutzte Variante
    • Obermenge zur CSS3-Syntax
    • d.h. jede gültige CSS3-Syntax ist gültige SCSS-Syntax.
    • Dateiendung: .scss
  • Funktional sind SASS und SCSS gleich.
  • Wir betrachten im folgenden nur SCSS!

Nachteile von SASS

  • Bisher unterstützen nur wenige Editoren SASS/SCSS
  • Direkte Änderungen an der .css datei sind nicht mehr möglich, da diese nach Änderung der .scss überschrieben werden
  • Debugging umständlicher (z.B. stimmt die Zeilenangabe in Firebug nicht mit .sass Datei überein)
  • Neue potenzielle Fehlerquellen
  • Die Direktheit mit der man HTML und CSS bearbeiten kann, geht duch den Kompilierungs-Schritt etwas verloren
  • Kaskadierungsgedanke von CSS geht verloren

Wie kann ich SASS benutzen

Variablen


// Farben
$textcolor:#4f4f4f;
$navcolor:#DFE1CB;
// Fonts
$textfont:arial, helvetica, san-serif;
$fontsize:12px;
// Sizes
$marginleft:90px;
$marginright:$marginleft + 5px;
  • Werden mit $VARNAME: VALUE; definiert.
  • Mit numerischen Variablen kann gerechnet werden. (+, -, *, /)

Nesting / Verschachtelung


nav {
    width:330px;
    margin-left:0;
    margin-right:0;
    ul {
        list-style-type:none;
        margin:0;
        padding:0;
    }
    li {
        float:left;
        background-color:$navcolor;
        margin:0px;
        padding:0px;
    }
}
  • Tags können wie im HTML verschachtelt werden.

Nesting / Verschachtelung

Aus vorangegangener .scss ensteht folgende .css 


nav {
    width:330px;
    margin-left:0;
    margin-right:0; 
}
nav ul {
    list-style-type:none;
    margin:0;
    padding:0; 
}
nav li {
    float:left;
    background-color:#DFE1CB;
    margin:0px;
    padding:0px; 
}

Mixins


@mixin rounded-corners {
    border-width:1px;
    border-color:#eeeffd;
    -moz-border-radius:5px;
    -webkit-border-radiu:5px;
}

div.somecontainer {
    @include rounded-corners;
}
div.anothercontainer {
    @include rounded-corners;
}
  • Eigenschaften können mit @mixin definiert und mit @include wiederverwendet werden
  • Praktisch für Standardfomatierungen

Mixins

Aus vorangegangener .scss ensteht folgende .css 


div.somecontainer {
  border-width: 1px;
  border-color: #eeeffd;
  -moz-border-radius: 5px;
  -webkit-border-radiu: 5px; 
}
div.anothercontainer {
  border-width: 1px;
  border-color: #eeeffd;
  -moz-border-radius: 5px;
  -webkit-border-radiu: 5px; 
}

Mixins mit Parameter


@mixin rounded-corners($width, $color, $radius) {
    border-width:$width;
    border-color:$color;
    -moz-border-radius:$radius;
    -webkit-border-radiu:$radius;
}

div.somecontainer {
    @include rounded-corners(1px, #eeeffd, 5px);
}
div.anothercontainer {
    @include rounded-corners(2px, #ffeed4, 8px);
}
  • Zusätzlich können Parameter angegeben werden
  • Vergleichbar mit Funktionen aus üblichen Programmiersprachen

Mixins mit Parameter

Aus vorangegangener .scss ensteht folgende .css 


div.somecontainer {
  border-width: 1px;
  border-color: #eeeffd;
  -moz-border-radius: 5px;
  -webkit-border-radiu: 5px; 
}
div.anothercontainer {
  border-width: 2px;
  border-color: #ffeed4;
  -moz-border-radius: 8px;
  -webkit-border-radiu: 8px; 
}

Vererbung


h1 {
    font-size:30px;
    font-weight:bold;
    color:#fffdde;
}
h2 {
    @extend h1;
    font-size:20px;
}
  • Mittels @extend können Eigenschaften ganzer Selektoren "vererbt" werden

Vererbung

Aus vorangegangener .scss ensteht folgende .css 


h1, h2 {
  font-size: 30px;
  font-weight: bold;
  color: #fffdde; 
}
h2 {
  font-size: 20px; 
}

1. Aufgabe

Als ertes müssen wir in einem Ordner eine datei style.scss überwachen, damit diese bei Änderung automatisch zu einer style.css übersetzt wird.
Hierzu installieren wir Ruby und Sass.
(Die style.scss mit der enstsprechenden index.html findet ihr im Ordner uebung (CSS-Frameworks branch))

 

1.1 Windows-user:

Ruby installieren: klick
Ruby-Terminal öffnen: Start -> “ruby” -> Start Command Prompt with Ruby

 

1.2 Windows- und Mac-user (mac OS hat ruby vorinstalliert):

Sass installieren: “gem install sass” in terminal eingeben. Zum Ziel-Ordner wechseln.
Datei überwachen mit “sass --watch style.scss:style.css” in terminal eingeben.

Wenn alles geklappt hat wurde jetzt eine style.css erstellt, die bei jeder Änderung der style.scss aktualisiert wird. ACHTUNG: Terminal offen lassen!

1.3 Im Notfall:

Falls es Probleme gibt oder einem das Terminal garnicht zusagt, könnt ihr folgende Seite nutzen um SCSS zu übersetzen.

Jedoch nur für Testzwecke geeignet klick
