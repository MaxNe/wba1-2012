Teil 1: Was ist SASS?

Was ist SASS?

  • 2007 von Hampton Catlin, Nathan Weizenbaum und Chris Eppstein eigeführt
  • SASS ist eine Metasprache zu CSS
    • d.h. SASS-Dateien werden in standard CSS-Dateien übersetzt
  • Erweitert CSS3 mit "programmiertechnischen" Funktionen wie:
    • Variablen
    • Nesting (Verschachtelung)
    • Mixins
    • Vererbung

 

 

Teil 2: Warum SASS?

Warum SASS/SCSS?

Nachteile von normalem CSS

  • Eigenschaften wiederholen sich häufig
  • Umständlich bei Änderungen von Angaben, welche mehrfach an verschiedenen stellen auftauchen
  • Elemente besitzen oft dieselben Deklarationen mit gleichen Werten.
  • Selektoren können sehr lang und unübersichtlich werden.

Warum SASS/SCSS?

Vorteile und Fähigkeiten von SASS

  • Angaben können zentral definiert werden. (z.B. Farben, Abstände etc.)
  • SASS kann rechnen (z.B. Farben und Abstände relativ zu einem Wert berechnen)
  • Übersichtlicher und strukturierter
    • Selektoren können andere Eigenschaften erben
    • Selektoren können geschachtelt werden
  • Folglich: erhöhte Wartbarkeit

 

 

Teil 3: Unterschiede zwischen SASS und SCSS

SASS gibts es in zwei Dialekten: 1. SASS und 2. SCSS

  • SASS - Syntactically Awesome Stylesheet
    • Ältere Variante
    • Verzichtet auf Klammern und Semikolons
    • Achtet stattdessen auf die Einrückung des Codes
    • Dateiendung: .sass

SASS gibts es in zwei Dialekten: 1. SASS und 2. SCSS

  • SCSS - Sassy Cascading Stylesheet
    • Neuere und zumeist genutzte Variante
    • Obermenge zur CSS3-Systax
    • d.h. jede gültige CSS3-Syntax ist gültige SCSS-Syntax.
    • Dateiendung: .scss
  • Funktional sind SASS und SCSS gleich.
  • Wir betrachten im folgenden nur SCSS!

Kurzes Code-Beispiel

SASS-Syntax                                  SCSS-Syntax


$blue: #3bbfce                              $blue: #3bbfce;
$margin: 16px                               $margin: 16px;

.border                                     .border {
  padding: $margin / 2                        padding: $margin / 2;
  margin: $margin / 2                         margin: $margin / 2;
  border-color: $blue                         border-color: $blue;
                                            }

 

 

Teil 4: Wie kann ich SASS/SCSS benutzen?

Variablen


// Farben
$textcolor:#4f4f4f;
$navcolor:#DFE1CB;
// Fonts
$textfont:arial, helvetica, san-serif;
$fontsize:12px;
// Sizes
$marginleft:90px;
$marginright:$marginleft + 5px;
  • Werden mit $VARNAME: VALUE; definiert.
  • Mit numerischen Variablen kann gerechnet werden. (+, -, *, /)

Nesting / Verschachtelung


nav {
    width:330px;
    margin-left:0;
    margin-right:0;
    ul {
        list-style-type:none;
        margin:0;
        padding:0;
    }
    li {
        float:left;
        background-color:$navcolor;
        margin:0px;
        padding:0px;
    }
}
  • Tags können wie im HTML verschachtelt werden.

Nesting / Verschachtelung

Aus vorangegangener .scss ensteht folgende .css 


nav {
    width:330px;
    margin-left:0;
    margin-right:0; 
}
nav ul {
    list-style-type:none;
    margin:0;
    padding:0; 
}
nav li {
    float:left;
    background-color:#DFE1CB;
    margin:0px;
    padding:0px; 
}

Mixins


@mixin rounded-corners {
    border-width:1px;
    border-color:#eeeffd;
    -moz-border-radius:5px;
    -webkit-border-radiu:5px;
}

div.somecontainer {
    @include rounded-corners;
}
div.anothercontainer {
    @include rounded-corners;
}
  • Eigenschaften können mit @mixin definiert und mit @include wiederverwendet werden
  • Praktisch für Standardfomatierungen

Mixins

Aus vorangegangener .scss ensteht folgende .css 


div.somecontainer {
  border-width: 1px;
  border-color: #eeeffd;
  -moz-border-radius: 5px;
  -webkit-border-radiu: 5px; 
}
div.anothercontainer {
  border-width: 1px;
  border-color: #eeeffd;
  -moz-border-radius: 5px;
  -webkit-border-radiu: 5px; 
}

Mixins mit Parameter


@mixin rounded-corners($width, $color, $radius) {
    border-width:$width;
    border-color:$color;
    -moz-border-radius:$radius;
    -webkit-border-radiu:$radius;
}

div.somecontainer {
    @include rounded-corners(1px, #eeeffd, 5px);
}
div.anothercontainer {
    @include rounded-corners(2px, #ffeed4, 8px);
}
  • Zusätzlich können Parameter angegeben werden
  • Vergleichbar mit Funktionen aus üblichen Programmiersprachen

Mixins mit Parameter

Aus vorangegangener .scss ensteht folgende .css 


div.somecontainer {
  border-width: 1px;
  border-color: #eeeffd;
  -moz-border-radius: 5px;
  -webkit-border-radiu: 5px; 
}
div.anothercontainer {
  border-width: 2px;
  border-color: #ffeed4;
  -moz-border-radius: 8px;
  -webkit-border-radiu: 8px; 
}

Vererbung


h1 {
    font-size:30px;
    font-weight:bold;
    color:#fffdde;
}
h2 {
    @extend h1;
    font-size:20px;
}
  • Mittels @extend können Eigenschaften ganzer Selektoren "vererbt" werden

Vererbung

Aus vorangegangener .scss ensteht folgende .css 


h1, h2 {
  font-size: 30px;
  font-weight: bold;
  color: #fffdde; 
}
h2 {
  font-size: 20px; 
}

  • Nesting Properties
  • Ausgelagerte Datei einbinden @include
  • Kontrollstrukturen @if
  • Schleifen @for
  • Funktionen (mit Rückgabewert)

Teil 6: Nachteile von SASS

Nachteile von SASS

  • Bisher unterstützen nur wenige Editoren SASS/SCSS
  • Direkte Änderungen an der .css datei sind nicht mehr möglich, da diese nach Änderung der .scss überschrieben werden
  • Debugging umständlicher (z.B. stimmt die Zeilenangabe in Firebug nicht mit .sass Datei überein)
  • Neue potenzielle Fehlerquellen
  • Die Direktheit mit der man HTML und CSS bearbeiten kann, geht duch den Kompilierungs-Schritt etwas verloren
  • Kaskadierungsgedanke von CSS geht verloren
  • SASS muss auf der Maschine erst installiert werden
