Was ist ein Player / Plugin

  • Benötigt um Videos im Internet abzuspielen
  • Werden in Form eines Plugins in den Browser integriert
  • Beachten: Spielen unterschiedliche Formate ab
  • HTML5 könnte Plugins überflüssig machen

Liste von Playern

  • Wohl bekannteste: Flash wird als Plugin in den Browser integriert, spielt nur Flash ab
  • Silverlight von Microsoft. Unterstüzt mehr Formate als Flash. Ebenso ein Plugin
  • JWPlayer Open Source und unterstüzt sowohl Flash als auch HTML5. Wird Serverseitig installiert, grundlage jedoch Flash
  • VideoJS Ebenso ein Open Source Player der Flash und HTML5 unterstüzt. Komplett kostenlos, wird Serverseitig installiert. Kein Plugin erforderlich

HTML5 Player

  • Alternative gegenüber Plugins
  • definiert das video Element
  • kein Videoformat definiert, nicht jeder Browser spielt das gleiche Format ab
  • daher kein vollständiger Ersatz für Plugins

HTML5 - Vorteile

  • video und audio, erlauben Video- als auch Audiodateien direkt in HTML-Dokumente einzubetten
  • HTML, CSS und JavaScript
  • Per CSS-Definitionen kann das Erscheinungsbild des Videos angepasst werden
  • Per CSS und SVG sogar Videofilter (Farbfilter, Unschärfe etc.) möglich
  • In meisten Browsern bereits (wenn auch unvollständig) implementiert
  • Suchmaschinenfreundlich über Metadaten
  • In Zukunft universal auf allen Endgeräten und mit Inhalt in allen Weltsprachen verwendbar

HTML5 - Nachteile

  • Nicht alle HTML5-Funktionalitäten in allen Browsern verfügbar
  • Kein Standard für Audio/Video-Codecs  wie eigentlich geplant
  • Mangel an Zugänglichkeit über Screenreader
  • Noch keine ausgereiften und stabilen Entwicklungsumgebungen verfügbar
  • JavaScript bietet noch keine ausgereifte 3D-Technologie
  • Erhöhter Aufwand über mehrere Versionen
  • Anhaltende Cross-Browser-Problematik auch mit CSS3

Flash - Vorteile

  • Über Plugin keine Probleme mit Browserkompatibilität
  • Effektivere Cross-Browser-Entwicklung
  • Einfaches Einbetten von Multimedia-Content wie Animationen, Audio, Video
  • 3D-Unterstützung ausgereift
  • Auslesen und Ansteuern in Echtzeit, Multi- Touch-Anwendungen, Motion-Steuerung möglich

Flash - Nachteile

  • Plugininstallation in Form von Flashplayer nötig
  • Mangelnde Performance für Nicht-Windows-Nutzer
  • Sicherheitslücken über Plugin
  • Kein Flashplayer für iPhone/iPad
  • Auffindbarkeit von Infos problematisch über kompiliertes swf-Format: Suchmaschinenunfreundlich

HTML5 Video Player

Vimeo oder Youtube

  • Videoportale generieren automatisch ein Einbettungscode
  • Einfache Copy/Paste Aktion
Beispielvideo: 

<iframe  width="320" height="240" src="http://player.vimeo.com/video/48399328?title=0&byline=0&portrait=0&color=e2007a" frameborder="0" </iframe>
  • iFrame wirkt sich positiv auf die Ladezeit der Seite aus
  • Damit ist ein paralleles laden der Inhalte möglich und die Seite wird komplett angezeigt

HTML5

Anfangs wird der <video> und am Ende der </video> Tag gesetzt. Dazwischen genügt die Quelle: 
<source src="wwww.InternetAdresse.de/Film.mp4" type="video/mp4">

                    <video width="320" height="240" controls>
                    <source src="movie.mp4" type="video/mp4">
                    <source src="movie.ogg" type="video/ogg">
                    </video>

Wichtige Parameter für das video Tag

  • width / height
  • src
  • type
  • controls
  • autoplay
  • looped

VideoJS - Head

  • Eine Javascript Bibliothek
  • Muss in den <head>eingebunden werden
<Head> 

<link href="http://vjs.zencdn.net/c/video-js.css" rel="stylesheet">
<script src="http://vjs.zencdn.net/c/video.js"></script>

VideoJS - Body

<Body> 

<video id="my_video_1" class="video-js vjs-default-skin" controls
width="640" height="264" poster="my_video_poster.png"
data-setup="{}">
<source src="my_video.mp4" type='video/mp4'>
<source src="my_video.webm" type='video/webm'></video>
  • Vorteil: alles im <Video> Tag integriert
  • controls schaltet Bedienleiste ein
  • poster die Quelle des Vorschaubildes
  • Stichwort: Fallback Lösung
