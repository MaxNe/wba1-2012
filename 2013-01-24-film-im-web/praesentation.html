HTML5

Vorteile

  • video und audio, erlauben Video- als auch Audiodateien direkt in HTML-Dokumente einzubetten
  • HTML, CSS und JavaScript
  • Per CSS-Definitionen kann das Erscheinungsbild des Videos angepasst werden
  • Per CSS und SVG sogar Videofilter (Farbfilter, Unschärfe etc.) möglich
  • In meisten Browsern bereits (wenn auch unvollständig) implementiert
  • Suchmaschinenfreundlich über Metadaten
  • In Zukunft universal auf allen Endgeräten und mit Inhalt in allen Weltsprachen verwendbar

HTML5

Nachteile

  • Nicht alle HTML5-Funktionalitäten in allen Browsern verfügbar
  • Kein Standard für Audio/Video-Codecs  wie eigentlich geplant
  • Mangel an Zugänglichkeit über Screenreader
  • Noch keine ausgereiften und stabilen Entwicklungsumgebungen verfügbar
  • JavaScript bietet noch keine ausgereifte 3D-Technologie
  • Erhöhter Aufwand über mehrere Versionen
  • Anhaltende Cross-Browser-Problematik auch mit CSS3

Flash

Vorteile

  • Über Plugin keine Probleme mit Browserkompatibilität
  • Effektivere Cross-Browser-Entwicklung
  • Einfaches Einbetten von Multimedia-Content wie Animationen, Audio, Video
  • 3D-Unterstützung ausgereift
  • Auslesen und Ansteuern in Echtzeit, Multi- Touch-Anwendungen, Motion-Steuerung möglich

Flash

Nachteile

  • Plugininstallation in Form von Flashplayer nötig
  • Mangelnde Performance für Nicht-Windows-Nutzer
  • Sicherheitslücken über Plugin
  • Kein Flashplayer für iPhone/iPad
  • Auffindbarkeit von Infos problematisch über kompiliertes swf-Format: Suchmaschinenunfreundlich

Audio-Video-Funktionalitäten: Flash

  • Flashplayer-Plugin
  • fasst problemlos alle gängigen Video- und Audioformate zusammen
  • 2D-Vektorgrafiken, 2D/3D-Animationen und Interaktion
  • Grafische Werkzeuge und All-In-One-Entwicklungsumgebung
  • Visuelle, zeitleistenbasierte Anordnung von Bewegtbild und Audio

Audio-Video-Funktionalitäten: HTML5

  • HTML5 (ohne Plugin und zusätzliche Codecs)
  • Audio- und Video-Inhalte direkt aus Dokument referenzierbar
  • Oberfläche mit CSS einfach anpassbar - Flash-Videoplayer-Anpassungen oft schwierig
  • Im Browser verfügbar und direkt manipulierbar über JavaScript

Videos einbinden

HTML5

  • Anfangs wird der <video> und am Ende der </video> Tag gesetzt. Dazwischen genügt die Quelle: <source src="wwww.InternetAdresse.de/Film.mp4" type="video/mp4">
<video width="320" height="240" controls> <source src="movie.mp4" type="video/mp4"> <source src="movie.ogg" type="video/ogg"></video>

Wichtige Parameter für das video Tag

  • width&height
  • src
  • type
  • controls
  • autoplay
  • looped
