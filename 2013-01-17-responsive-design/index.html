Was ist Responsive Design?


Begriff geprägt von Ethan Marcotte, Autor des Buches „Responsive Web Design“:

„Rather than tailoring disconnected designs to each of an ever-increasing number of web devices, we can treat them as facets of the same experience. We can design for an optimal viewing experience, but embed standards-based technologies into our designs to make them not only more flexible, but more adaptive to the media that renders them.“

Ethan Marcotte

"ever-increasing number of web devices"

Steigende Anzahl von Smartphones, Tablets, und alle mit verschiedenen Displaygrößen.

"design for an optimal viewing experience"

"Das Auge isst mit" - Eine Seite für alle Geräte.

"not only more flexible, but more adaptive"

Anpassungsfähige und flexible Websites.

1. fluid grids

  • Flexible Raster, die sich an Breakpoints orientieren.
  • Sinnvolle Breakpoints:

mostly fluid

  • Bricht erst ab minimaler Breite zusammen.
  • Nachteil: breite Ränder auf größeren Bildschirmen.
  • Beispiel

column drop

  • Ähnlich zu "mostly fluid", aber bildschirmfüllend.
  • Die Spalten brechen nacheinander untereinander zusammen.

3. media queries

  • CSS3 media queries wird in einem stylesheet eingebunden und fragt bestimmte Eigenschaften ab.
  • Zum Beispiel den Viewport, die Display/Browser-Breite/Höhe, die Ausrichtung (Portrait/Landscape) und Auflösung.

3.1 Eigenschaften

    screen:
  • Für Bildschirme
    • max-width:
  • Maximale Breite des Fenster
    • max-device-width:
  • für Smartphones/Tablets
    • orientation: landscape
    print:
  • Für Dokumente
    • min-width:
  • minimale breite des Fensters
    • min-device-width:
  • für Smartphones/Tablets
    • orientation: portrait

3.2 Beispiel

@media screen and (max-width: 960px) { 
/* Hier steht der Code. */
}

Die Pixelangaben stellen den Breakpoint dar.

Im Funktionsblock können verschiedene Anweisungen stehen:

@media screen and (max-width: 960px) { 
    div#teaser {
        height: 200px;
    }
        
    #wrapper h1 {
        color: #aaaaa
    }
}
